VC调用COM组件

发表于 2020-05-11 分类于 vc
本文字数： 764 阅读时长 ≈ 1 分钟

COM技术是微软的一个通用组件框架技术，支持跨语言调用，只要遵循COM规则调用即可。com支持的文件形式也是多种多样，dll，ocx，tlb等等。
ActiveX控件是基于COM技术的一种，visual studio通常在“工具”->”选择工具箱” 即可看到系统提供了丰富的COM组件，如下图各种语言框架下的都有。
下面我们以调用excel和ActiveX控件为例。

excel调用

选择右击工程，选择”类向导”，在“添加类”处，选择“类型库中的MFC类”。
或者在“添加类”中，添加相应的MFC类，如下图：

点击“添加”。由于excel我们需要从安装的excel.exe中导入，故我们选择“从文件”导入形式，生成相应的类文件（*.h,*.cpp）。

后续的excel操作就是很简单的接口调用了，本文不做赘述。
excel的通用操作类是：App->worksheets->worksheet->ranges->range->cell(对应单元格)
若涉及图表的操作还会涉及如下的几个类：ChartObjects、ChartObject、_Chart、Series.

windows media play调用

对于wmp，微软也提供了一个ActiveX控件。该控件支持对话框设计界面的控件拖拽，但是vs默认在“工具箱”中是不显示的，我们需要先在菜单“工具”中“选择工具箱”，找到COM组件中的windows media play，勾选，然后确定，我们即可在“工具箱”中看到了。
至此，如果我们直接给控件静态绑定路径已经可以正常播放视频了，但是如果我们需要动态调用，设置播放列表，控制音频大小，进度条等等。我们必须“添加类”，然后wmp.dll导入,生成所有相关的类(CWMPPlayer4,CWMPSetting)。如下图，即可。
